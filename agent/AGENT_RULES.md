# ArchSec Engineering Agent Rules

1. This project follows controlled development.
2. You must NEVER delete:
   - TODO.md
   - migrations
   - working features
3. You must NEVER remove TODO items.
4. You may only:
   - append new TODO tasks
   - mark completed ones with [x]
5. TODO.md must always contain sections:
   - ## Bugs
   - ## Features
   - ## Refactoring
   - ## Infrastructure
6. If any error occurs, consult TODO.md before making changes.
7. Never modify database schema unless explicitly requested.
8. Never run shell commands.
9. Never refactor unrelated code.
10. Always preserve working behavior.
11. Every fix must be documented in TODO.md.
